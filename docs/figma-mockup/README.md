# Figma-mockup в теме Sonar

HTML-каркасы ключевых экранов Syncwatch в теме **Sonar** (светлая,
deep-blue акцент). Сделаны для импорта в Figma через плагин
[html.to.design](https://www.figma.com/community/plugin/1159123024924461424/) —
после импорта плагин конвертирует разметку в Figma-фреймы, дальше с ними
можно работать как с обычным макетом.

## Файлы

| Файл                 | Экран                          | Ширина |
| -------------------- | ------------------------------ | ------ |
| `home-desktop.html`  | Лента видео (сетка 4-в-ряд)    | 1440   |
| `home-mobile.html`   | Лента видео (один столбец)     | 360    |
| `watch-desktop.html` | Страница видео с плеером       | 1440   |
| `watch-mobile.html`  | Страница видео (мобильный)     | 360    |
| `room-desktop.html`  | Комната + сайд-панель чата     | 1440   |
| `room-mobile.html`   | Комната (плеер + чат сверху-вниз) | 360 |
| `auth-desktop.html`  | Вход/регистрация + sonar-ping  | 1440   |
| `auth-mobile.html`   | Вход/регистрация (мобильный)   | 360    |

## Технические детали

- Цвета — токены Sonar: белый фон, deep-blue (#1E3A8A) акцент, прохладные
  серые эшелоны для поверхностей.
- Шрифты — `Inter` (UI) и `JetBrains Mono` (логотип, таймкоды), подгружаются
  через Google Fonts CDN.
- Каждый файл самодостаточен — стили и SVG-иконки inline, без зависимостей
  кроме CDN-шрифтов.
- Pixel-perfect фиксированные ширины (1440 / 360) — для аккуратного импорта
  в Figma.

## Импорт в Figma

1. Открыть Figma → запустить плагин **html.to.design**.
2. На вкладке «URL» вставить ссылку на raw-файл из GitHub
   (`https://raw.githubusercontent.com/BLXCKBXXST/syncwatch/main/docs/figma-mockup/<file>.html`)
   или скопировать содержимое локального файла во вкладку «Code».
3. Дождаться импорта — плагин создаст фрейм с правильной шириной.
4. После импорта — переименовать слои, объединить парные desktop/mobile
   фреймы в страницы по экранам.
